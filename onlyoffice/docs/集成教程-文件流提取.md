# ONLYOFFICE 离线版集成教程（含文件流提取）

> 适用于本仓库的离线 WASM 版 ONLYOFFICE（无文档服务器、基于 `x2t.wasm` 客户端转换）。
> 本文重点讲清**如何把编辑后的文档以文件流（字节）方式取出**，交给你自己的后端去保存。

---

## 1. 架构与核心难点

### 1.1 这套离线版的特点

- 整个 ONLYOFFICE 跑在浏览器里，**没有 Document Server / 协同服务器**。
- 文档的打开、编辑、导出全部在客户端用 `x2t.wasm` 完成。
- 编辑器通过 `DocsAPI.DocEditor` 创建，运行在一个 iframe 里。

### 1.2 难点：离线版拿不到「保存回调」

常规 ONLYOFFICE 的保存依赖文档服务器：

- `editorConfig.callbackUrl`：文档服务器在保存时把文件 POST 给你的后端。
- `docEditor.downloadAs()` → `onDownloadAs` 事件：返回一个**服务器上的文件 URL**。

**这两条在离线版都不可用**（没有服务器）。离线版点保存 / 导出时，实际链路是：

```
docEditor.downloadAs('docx')
  └─ baseEditorsApi.downloadAs()                 (offline.js 重写)
       └─ _downloadAsFromLocal()
            └─ AscCommon.x2t.convertFromBin(...)  (WASM 客户端转换)
                 └─ result.binary                 ← 这就是文件字节！
                      └─ AscCommon.x2t.downloadFile(binary, fileName)
                           └─ 创建 <a download> → 触发浏览器下载
```

所以默认表现是「**只会触发浏览器下载，拿不到字节、也没有回调**」。

> 关键洞察：文件字节就在 `x2t.downloadFile(data, fileName)` 的第一个参数 `data` 里。
> **文件流提取 = 劫持 `x2t.downloadFile`，把 `data` 抛给宿主，并抑制浏览器下载。**

---

## 2. 三层窗口结构

集成时通常是三层 iframe（自上而下）：

```
业务页面 (你的 App)                       ← 接收文件流、交给后端
   └─ onlyoffice.html (包装层/宿主)        ← 注入配置、转发消息、声明只要文件流
        └─ DocsAPI 编辑器 iframe           ← 运行 sdk + x2t_helper.js
```

- **业务页面**：你的前端，负责发起保存、拿到文件流后交给后端。
- **onlyoffice.html**：薄薄一层包装，`new DocsAPI.DocEditor(...)`，并在业务页面与编辑器之间转发 `postMessage`。
- **编辑器 iframe**：ONLYOFFICE 自身，`x2t_helper.js` 在这一层运行。

消息全部用 `postMessage` 串起来。

---

## 3. 文件流提取：改造 `x2t.downloadFile`

文件：`vendor/sdkjs/common/wasm/x2t/x2t_helper.js`，`X2TConverter.prototype.downloadFile`。

把原本「创建 `<a download>` 下载」的实现，改成「**先把字节 postMessage 给宿主**，宿主声明只要流时就跳过下载」：

```js
X2TConverter.prototype.downloadFile = function (data, fileName) {
    // ① 对外提供文件流：把导出的文件字节 postMessage 给宿主窗口（父窗口/顶层）
    try {
        var buffer;
        if (data instanceof ArrayBuffer) {
            buffer = data.slice(0);
        } else if (data && data.buffer instanceof ArrayBuffer) {
            buffer = data.buffer.slice(data.byteOffset || 0, (data.byteOffset || 0) + data.byteLength);
        } else if (data) {
            buffer = new Uint8Array(data).buffer;
        }
        if (buffer) {
            var ext = (String(fileName || '').split('.').pop() || '').toLowerCase();
            var payload = { type: 'onlyoffice-file-stream', fileName: fileName, fileType: ext, buffer: buffer };
            var targets = [];
            if (window.parent && window.parent !== window) targets.push(window.parent);
            if (window.top && window.top !== window && window.top !== window.parent) targets.push(window.top);
            targets.forEach(function (t) {
                try { t.postMessage(payload, '*'); } catch (e) {}
            });
        }
    } catch (e) {}

    // ② 宿主声明只要文件流时跳过浏览器下载（沿父窗口链查找标志，兼容多层 iframe 嵌套）
    var __ooStreamOnly = false;
    try {
        for (var __w = window, __d = 0; __w && __d < 6; __d++) {
            if (__w.OO_FILE_STREAM_ONLY === true) { __ooStreamOnly = true; break; }
            if (__w.parent === __w) break;
            __w = __w.parent;
        }
    } catch (e) {}
    if (__ooStreamOnly) {
        return;
    }

    // ③ 默认行为：浏览器下载（独立 demo / 未声明只要流时仍可用）
    var blob = new Blob([data]);
    var url = URL.createObjectURL(blob);
    var link = document.createElement('a');
    link.href = url;
    link.download = fileName;
    link.style.display = 'none';
    document.body.appendChild(link);
    link.click();
    setTimeout(function () {
        document.body.removeChild(link);
        URL.revokeObjectURL(url);
    }, 100);
};
```

要点：

- **始终**把文件流 `postMessage` 出去（`type: 'onlyoffice-file-stream'`，`buffer` 是 `ArrayBuffer`）。
- 用结构化克隆（不 transfer），每个目标窗口各拿一份拷贝。
- `OO_FILE_STREAM_ONLY` 标志**沿父窗口链查找**：编辑器自身可能没设，但它的父窗口（onlyoffice.html）设了就算数——比「从外面 reach-in 设编辑器 window」更可靠，且兼容多层 iframe。
- 未声明该标志的场景（如独立 `office.html` demo）→ 仍正常浏览器下载，不受影响。

## 4. 宿主：声明只要流 + 触发保存 + 转发文件流

文件：`onlyoffice.html`（包装层）。

### 4.1 声明只要文件流（脚本最顶部）

```js
// 本宿主只要文件流、不要浏览器下载。内层编辑器(x2t.downloadFile)会沿父窗口链读取此标志。
window.OO_FILE_STREAM_ONLY = true;
```

### 4.2 创建编辑器时注册 `onDownloadAs`（空实现）

`downloadAs` 方法要求声明了 `onDownloadAs` 才会执行；离线版实际不回调它（文件由劫持的 `downloadFile` 捕获），所以放一个空函数即可：

```js
docConfig.events = docConfig.events || {};
docConfig.events.onDocumentReady = onDocumentReady;
docConfig.events.onDownloadAs   = function () {};   // 仅为放行 downloadAs
docEditor = new DocsAPI.DocEditor('editor', docConfig);
```

### 4.3 收到保存指令 → 触发 downloadAs

```js
var pendingSave = null;

function postToParent(msg, transfer) {
    if (window.parent && window.parent !== window) {
        window.parent.postMessage(msg, '*', transfer || []);
    }
}

// 业务页面请求保存：触发 downloadAs，文件由 x2t.downloadFile 以流抛出
function handleSaveCommand(data) {
    if (!docEditor) {
        postToParent({ type: 'onlyoffice-saved', requestId: data && data.requestId, ok: false, error: '编辑器未就绪' });
        return;
    }
    pendingSave = { requestId: data && data.requestId, format: data && data.format };
    try {
        docEditor.downloadAs(data && data.format ? data.format : undefined);  // 如 'docx' / 'xlsx'
    } catch (e) {
        pendingSave = null;
        postToParent({ type: 'onlyoffice-saved', requestId: data && data.requestId, ok: false, error: String((e && e.message) || e) });
    }
}
```

### 4.4 接收编辑器抛出的文件流 → 转发给业务页面

```js
window.addEventListener('message', function (event) {
    var data = event.data;
    if (!data) return;

    if (data.type === 'onlyoffice-config' && data.docConfig) {
        loadOnlyoffice(typeof data.docConfig === 'string' ? JSON.parse(data.docConfig) : data.docConfig);
    } else if (data.type === 'onlyoffice-save') {
        handleSaveCommand(data);                 // 来自业务页面：请求保存
    } else if (data.type === 'onlyoffice-file-stream') {
        handleFileStream(data);                  // 来自编辑器：抛出的文件流
    }
});

// 把文件流关联到当前保存请求，回传业务页面
function handleFileStream(data) {
    var req = pendingSave;
    pendingSave = null;
    var buffer = data && data.buffer;
    if (!buffer) {
        postToParent({ type: 'onlyoffice-saved', requestId: req && req.requestId, ok: false, error: '未获取到文件流' });
        return;
    }
    postToParent({
        type: 'onlyoffice-saved',
        requestId: req && req.requestId,
        ok: true,
        buffer: buffer,                          // ArrayBuffer
        fileType: data.fileType || '',
        fileName: data.fileName || ''
    }, [buffer]);                                // 可转移：自己这份拷贝直接 transfer 给父窗口
}
```

> 编辑器 `downloadFile` 会把流同时发给 `window.parent`(onlyoffice.html) 和 `window.top`(业务页面)。
> 业务页面只处理 `onlyoffice-saved`、忽略 `onlyoffice-file-stream`，避免重复处理。

## 5. 业务页面：发起保存 + 拿到字节

完整握手时序：

```
业务页面 ──(onlyoffice-save {requestId, format})──▶ onlyoffice.html
onlyoffice.html ── docEditor.downloadAs(format) ──▶ 编辑器
编辑器 ── x2t 转换 → x2t.downloadFile(binary) ── 劫持 ──▶ postMessage(onlyoffice-file-stream)
onlyoffice.html ──(onlyoffice-saved {requestId, buffer})──▶ 业务页面
业务页面 ── new Blob([buffer]) → 交给你的后端
```

业务页面（原生 JS 示例）：

```js
const frame = document.querySelector('#oo-frame');   // 指向 onlyoffice.html 的 iframe
let saveSeq = 0;
const pending = {};

window.addEventListener('message', (e) => {
    const d = e.data;
    if (!d || d.type !== 'onlyoffice-saved') return;
    const p = pending[d.requestId];
    if (!p) return;
    delete pending[d.requestId];
    if (d.ok && d.buffer) {
        p.resolve(new Blob([d.buffer], { type: mimeOf(d.fileType) }));
    } else {
        p.reject(new Error(d.error || '保存失败'));
    }
});

// 触发保存，拿到编辑后的文件 Blob
function save(format = 'docx') {
    const requestId = 'save-' + (++saveSeq);
    return new Promise((resolve, reject) => {
        const timer = setTimeout(() => { delete pending[requestId]; reject(new Error('保存超时')); }, 20000);
        pending[requestId] = { resolve: (b) => { clearTimeout(timer); resolve(b); }, reject };
        frame.contentWindow.postMessage({ type: 'onlyoffice-save', requestId, format }, '*');
    });
}

function mimeOf(ext) {
    return ({
        docx: 'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
        xlsx: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
        pptx: 'application/vnd.openxmlformats-officedocument.presentationml.presentation'
    })[ext] || 'application/octet-stream';
}

// 用法：拿到 Blob 后，怎么存由你决定（上传后端 / 本地保存 / 再加工）
const blob = await save('docx');
// 例如上传：
const fd = new FormData();
fd.append('file', new File([blob], 'document.docx', { type: blob.type }));
await fetch('<你的上传接口>', { method: 'POST', body: fd });
```

> **加载文档**：通过 `postMessage({type:'onlyoffice-config', docConfig})` 注入到 onlyoffice.html，
> `docConfig.document.url` 指向一个可访问的文件 URL（本地 blob / 后端直链 / 临时签名 URL 均可）。
> 新建空文档时，准备一个空白 `blank.docx` / `blank.xlsx` 作为基底 `url`。
