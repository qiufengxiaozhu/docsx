## 1. 三种使用方式

### 方式一：独立使用（office.html）

直接打开 `office.html`，自带文件选择/新建界面，编辑后通过浏览器下载保存。适合个人离线使用，无需任何集成。

```bash
python -m http.server 8000
# 访问 http://localhost:8000/office.html
```

### 方式二：URL 参数注入配置（onlyoffice.html?docConfig=...）

把 `docConfig` 序列化后拼在 URL 上，适合简单嵌入、文档 URL 较短的场景：

```js
const docConfig = {
    document: {
        url: 'http://example.com/files/demo.docx',
        title: '演示文档.docx'
    }
}
const src = `/onlyoffice.html?docConfig=${encodeURIComponent(JSON.stringify(docConfig))}`
// <iframe :src="src"></iframe>
```

> ⚠️ 局限：URL 有长度限制（浏览器约 2000~8000 字符）。MinIO/S3 的 presigned URL 很容易超长，此时请用方式三。

### 方式三：iframe + postMessage（推荐的集成方式）

宿主页面嵌入 `onlyoffice.html`，通过 `postMessage` 注入配置、触发保存、接收文件流。这是功能最完整的方式，支持本文第 3、4 节的全部协议。

握手时序：

```
宿主页面                          onlyoffice.html                    编辑器
   │◀──── onlyoffice-ready ─────────│  (iframe 加载完成)
   │───── onlyoffice-config ───────▶│── new DocsAPI.DocEditor ──────▶│
   │◀─ onlyoffice-document-ready ───│◀───── onDocumentReady ─────────│
   │───── onlyoffice-save ─────────▶│────── downloadAs() ───────────▶│
   │                                │◀── onlyoffice-file-stream ─────│ (x2t 转换出字节)
   │◀──── onlyoffice-saved ─────────│
   │  new Blob(buffer) → 交给后端
```

最小示例（原生 JS）：

```html
<iframe id="oo" src="/onlyoffice.html" style="width:100%;height:100vh;border:0"></iframe>
<script>
const frame = document.getElementById('oo')

window.addEventListener('message', (e) => {
    const d = e.data
    if (!d) return
    if (d.type === 'onlyoffice-ready') {
        // iframe 就绪，注入文档配置
        frame.contentWindow.postMessage({
            type: 'onlyoffice-config',
            docConfig: {
                document: {
                    url: 'http://example.com/files/demo.docx',
                    title: '演示文档.docx'
                },
                editorConfig: { mode: 'edit', lang: 'zh-CN' }
            }
        }, '*')
    } else if (d.type === 'onlyoffice-saved' && d.ok) {
        // 拿到编辑后的文件字节
        const blob = new Blob([d.buffer])
        // 上传后端 / 本地处理...
    }
})

// 触发保存
function save() {
    frame.contentWindow.postMessage(
        { type: 'onlyoffice-save', requestId: 'save-1', format: 'docx' }, '*')
}
</script>
```

## 2. docConfig 配置说明

`docConfig` 与官方 ONLYOFFICE 的 [DocEditor 配置](https://api.onlyoffice.com/editors/config/)同构，常用字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `document.url` | String | ✅ | 文档地址（后端直链 / presigned URL / blob URL 均可，需可跨域访问） |
| `document.title` | String | 否 | 文档标题（含后缀），显示在编辑器顶部；不传则默认 `文档.<fileType>` |
| `document.fileType` | String | 否 | 文件后缀（docx/xlsx/pdf...）；不传则由 `url` 推断 |
| `document.key` | String | 否 | 文档唯一 key，影响编辑器缓存；不传则按 `url + title` 哈希生成。**同一文档的不同版本必须换 key** |
| `document.permissions` | Object | 否 | 权限：`{ edit: true, download: true, print: true }` |
| `documentType` | String | 否 | `word` / `cell` / `slide` / `pdf`；不传则由 `fileType` 推断 |
| `editorConfig.mode` | String | 否 | `edit` 编辑（默认）/ `view` 只读预览 |
| `editorConfig.lang` | String | 否 | 界面语言，如 `zh-CN` |
| `streamFallback` | String | 否 | 本项目扩展字段，见 [4.2](#42-自动保存--用户点下载streamfallback)。`autosave`（默认）/ `download` |

> `onlyoffice.html` 内置了配置规整逻辑：自动推断 `fileType` / `documentType`、生成 `key`、补全标题与尺寸，所以最少只需传 `document.url`。

## 3. postMessage 消息协议

### 宿主 → onlyoffice.html

| 消息 type | 字段 | 说明 |
|-----------|------|------|
| `onlyoffice-config` | `docConfig`（对象或 JSON 字符串） | 加载/切换文档。重复发送会销毁旧编辑器并重建 |
| `onlyoffice-save` | `requestId`、`format`（可选，如 `docx`/`xlsx`/`pdf`） | 触发保存，结果经 `onlyoffice-saved` 回传；`format` 可实现格式转换导出 |

### onlyoffice.html → 宿主

| 消息 type | 字段 | 触发时机 |
|-----------|------|----------|
| `onlyoffice-ready` | - | iframe 加载完成，等待注入配置 |
| `onlyoffice-document-ready` | - | 文档打开完成（`onDocumentReady`） |
| `onlyoffice-saved` | `requestId`、`ok`、`buffer`（ArrayBuffer）、`fileType`、`fileName`、`error` | 保存完成/失败。**无 `requestId` 时表示编辑器自动保存产生的流** |
| `onlyoffice-saveas` | `ok`、`buffer`、`fileType`、`error` | 用户在编辑器里点「另存为/保存副本」 |
| `onlyoffice-rename` | `title` | 用户在编辑器里重命名文档（`onRequestRename` / `onMetaChange`） |
| `onlyoffice-state-change` | `modified`（Boolean） | 文档修改状态变化（true = 有未保存修改） |
| `onlyoffice-request-close` | - | 用户点击编辑器自带的关闭按钮 |

## 4. 功能详解

### 4.1 保存文件流（核心功能）

离线版没有文档服务器，官方的 `callbackUrl` 保存回调不可用。本项目通过改造 `x2t_helper.js` 把导出字节以 `postMessage` 抛出（原理见[集成教程](集成教程-文件流提取.md)），宿主只需：

```js
let saveSeq = 0
const pending = {}

window.addEventListener('message', (e) => {
    const d = e.data
    if (!d || d.type !== 'onlyoffice-saved') return
    const p = pending[d.requestId]
    if (!p) return
    delete pending[d.requestId]
    d.ok && d.buffer ? p.resolve(new Blob([d.buffer])) : p.reject(new Error(d.error || '保存失败'))
})

function save(format = 'docx') {
    const requestId = 'save-' + (++saveSeq)
    return new Promise((resolve, reject) => {
        const timer = setTimeout(() => { delete pending[requestId]; reject(new Error('保存超时')) }, 20000)
        pending[requestId] = {
            resolve: (b) => { clearTimeout(timer); resolve(b) },
            reject: (err) => { clearTimeout(timer); reject(err) }
        }
        frame.contentWindow.postMessage({ type: 'onlyoffice-save', requestId, format }, '*')
    })
}

// 用法：拿到 Blob 后交给你的后端
const blob = await save('docx')
const fd = new FormData()
fd.append('file', new File([blob], '文档.docx', { type: blob.type }))
await fetch('/api/upload', { method: 'POST', body: fd })
```

要点：

- `requestId` 用于把保存请求和回传的文件流一一对应，支持并发/重试。
- `format` 即导出格式，**可与源格式不同**（如打开 docx、导出 pdf），由 `x2t.wasm` 在客户端完成转换。
- 保存是全量导出当前文档内容，与是否有修改无关。

### 4.2 自动保存 / 用户点下载（streamFallback）

编辑器内部的自动保存、用户主动点「下载」按钮，也会产生文件流。这类**没有对应保存请求**的流由 `docConfig.streamFallback` 决定去向：

| 值 | 行为 |
|----|------|
| `autosave`（默认） | 以无 `requestId` 的 `onlyoffice-saved` 回传宿主，宿主可静默上传实现自动保存 |
| `download` | 直接在 iframe 内触发浏览器下载，行为与官方版一致 |

```js
// 宿主区分主动保存和自动保存
if (d.type === 'onlyoffice-saved' && d.ok && !pending[d.requestId]) {
    uploadSilently(new Blob([d.buffer]))   // 自动保存 → 后台静默上传
}
```

### 4.3 文件名与重命名

- **指定文件名**：`document.title` 传入含后缀的完整文件名（如 `合同模板.docx`），显示在编辑器顶栏。
- **自动推断**：不传 `title` 时按 `url` 后缀生成默认名。
- **用户重命名**：用户在编辑器顶栏改名后，宿主收到 `onlyoffice-rename` 消息，可同步更新业务侧文件名：

```js
if (d.type === 'onlyoffice-rename') {
    renameFileOnServer(fileId, d.title)   // d.title 为新文件名
}
```

- **保存时的文件名**：`onlyoffice-saved` / 文件流消息携带 `fileName` 字段，为编辑器当前标题 + 导出后缀。

### 4.4 另存为（save-as）

用户在编辑器菜单里「保存副本」时，宿主收到 `onlyoffice-saveas` 消息，`buffer` 即副本文件字节：

```js
if (d.type === 'onlyoffice-saveas') {
    if (d.ok) saveCopyToServer(new Blob([d.buffer]), d.fileType)
    else console.error(d.error)
}
```

### 4.5 修改状态与关闭

```js
if (d.type === 'onlyoffice-state-change') {
    hasUnsavedChanges = d.modified        // 离开页面前提示"有未保存修改"
}
if (d.type === 'onlyoffice-request-close') {
    closeEditorTab()                      // 用户点了编辑器的关闭按钮
}
```

### 4.6 只读预览与权限

```js
{
    document: {
        url: fileUrl,
        permissions: { edit: false, download: true, print: true }
    },
    editorConfig: { mode: 'view' }
}
```

`mode: 'view'` 打开为只读预览；`permissions` 可进一步控制下载、打印按钮。

### 4.7 新建空文档

离线版新建文档 = 打开一个空白模板。仓库 `blank/` 目录提供了 `blank.docx` 和 `blank.xlsx`：

```js
{
    document: {
        url: '/blank/blank.docx',      // 空白模板作为基底
        title: '未命名文档.docx',
        key: 'new-' + Date.now()       // 新文档必须用新 key，避免命中缓存
    }
}
```

编辑后按 [4.1](#41-保存文件流核心功能) 正常保存即可得到新文件。

### 4.8 连接器（Automation API）

本产物的 `api.js` 内置了官方连接器（`docEditor.createConnector()` / `Asc.EditorConnector`），可在**不写插件**的情况下以编程方式操作文档：执行文档构建脚本、增删内容、读取选区、监听编辑器事件等。

连接器必须在持有 `docEditor` 实例的窗口（即 `onlyoffice.html`）中创建。在 `onlyoffice.html` 的 `onDocumentReady` 后使用：

```js
var connector = docEditor.createConnector()

// 用文档构建 API 在光标处插入文本
connector.callCommand(function () {
    var oDocument = Api.GetDocument()
    var oParagraph = Api.CreateParagraph()
    oParagraph.AddText('由连接器插入的文本')
    oDocument.InsertContent([oParagraph])
})

// 执行编辑器方法（如获取当前选中文本）
connector.executeMethod('GetSelectedText', [], function (text) {
    console.log('选中内容:', text)
})

// 监听编辑器事件
connector.attachEvent('onChangeContentControl', function (obj) {
    console.log('内容控件变化:', obj)
})
```

如需从宿主页面驱动连接器，可参照第 3 节协议自行扩展消息类型（如 `onlyoffice-command`），在 `onlyoffice.html` 中收到消息后调用 `connector.callCommand(...)` 并回传结果。

> `callCommand` 内的函数在编辑器上下文中执行，可用的 `Api` 对象参考官方 [Office API 文档](https://api.onlyoffice.com/docbuilder/basic)。

## 5. Vue 组件完整案例

以下是生产项目中使用的 Vue 2 组件封装（Vue 3 仅需把 `beforeDestroy` 改为 `beforeUnmount`），封装了握手、配置注入、保存 Promise 化、自动保存、另存为、重命名等全部协议。

部署方式：把仓库整体（`onlyoffice.html`、产物目录 `9.3.0.134-*`、`assets/`、`blank/`）复制到前端工程的 `public/onlyoffice/` 下。

使用：

```vue
<template>
  <only-office-viewer
    ref="viewer"
    :url="fileUrl"
    :title="fileName"
    mode="edit"
    height="80vh"
    @rename="onRename"
    @autosave="onAutosave"
    @state-change="onStateChange"
  />
</template>

<script>
export default {
  methods: {
    async saveToServer() {
      const blob = await this.$refs.viewer.save('docx')   // 触发保存，拿到文件流
      const fd = new FormData()
      fd.append('file', new File([blob], this.fileName, { type: blob.type }))
      await this.$http.post('/api/file/upload', fd)
    },
    onRename(title) { /* 同步文件名到后端 */ },
    onAutosave(blob, fileType) { /* 自动保存静默上传 */ },
    onStateChange(modified) { /* 未保存修改提示 */ }
  }
}
</script>
```

组件源码 `OnlyOfficeViewer.vue`：

```vue
<template>
  <div class="onlyoffice-viewer" :style="{ height: wrapHeight }">
    <iframe
      ref="frame"
      class="onlyoffice-frame"
      :src="frameSrc"
      frameborder="0"
      allowfullscreen
      @load="onFrameLoad"
    ></iframe>
  </div>
</template>

<script>
/**
 * OnlyOffice 通用预览/编辑组件（本地 WASM 版，无文档服务器）
 *
 * 用法：
 *   <only-office-viewer :url="fileUrl" :title="fileName" mode="view" />
 *
 * 文档通过 postMessage 注入 iframe，规避 MinIO presigned 长 URL 的长度限制。
 */
export default {
  name: 'OnlyOfficeViewer',
  props: {
    // 文档地址（后端 / MinIO 可访问的 url）
    url: { type: String, default: '' },
    // 文档标题；不传则由 url 推断
    title: { type: String, default: '' },
    // 文件后缀，如 docx/xlsx/pdf；不传则由 url 推断（iframe 内兜底）
    fileType: { type: String, default: '' },
    // 文档类型 word/cell/slide/pdf；不传则由 fileType 推断（iframe 内兜底）
    documentType: { type: String, default: '' },
    // edit 编辑 / view 只读预览
    mode: { type: String, default: 'edit' },
    // 唯一 key，影响协同/缓存；同一文档保持一致，不传则 iframe 内按 url+title 生成
    docKey: { type: String, default: '' },
    // 容器高度
    height: { type: [String, Number], default: '100%' },
    // 透传/覆盖完整 docConfig（高级用法），与上面 props 深合并
    config: { type: Object, default: () => ({}) }
  },
  data() {
    return {
      frameReady: false,
      documentReady: false,
      saveSeq: 0,
      pendingSaves: {}
    }
  },
  computed: {
    frameSrc() {
      // process.env.BASE_URL 即 vue-cli 的 publicPath，结尾带 '/'
      return `${process.env.BASE_URL}onlyoffice/onlyoffice.html`
    },
    wrapHeight() {
      return typeof this.height === 'number' ? `${this.height}px` : this.height
    }
  },
  watch: {
    url() {
      this.pushConfig()
    },
    mode() {
      this.pushConfig()
    },
    docKey() {
      this.pushConfig()
    },
    fileType() {
      this.pushConfig()
    }
  },
  mounted() {
    window.addEventListener('message', this.onMessage)
  },
  beforeDestroy() {
    window.removeEventListener('message', this.onMessage)
    // 拒绝所有未完成的保存，避免悬挂的 Promise
    Object.keys(this.pendingSaves).forEach(id => {
      const pending = this.pendingSaves[id]
      if (pending) {
        clearTimeout(pending.timer)
        pending.reject(new Error('编辑器已销毁'))
      }
    })
    this.pendingSaves = {}
  },
  methods: {
    onMessage(event) {
      const data = event.data
      if (!data) return
      if (data.type === 'onlyoffice-ready') {
        this.frameReady = true
        this.documentReady = false
        this.pushConfig()
        this.$emit('ready')
      } else if (data.type === 'onlyoffice-document-ready') {
        this.documentReady = true
        this.$emit('document-ready')
      } else if (data.type === 'onlyoffice-saved') {
        this.handleSaved(data)
      } else if (data.type === 'onlyoffice-request-close') {
        this.$emit('request-close')
      } else if (data.type === 'onlyoffice-state-change') {
        this.$emit('state-change', !!data.modified)
      } else if (data.type === 'onlyoffice-rename') {
        this.$emit('rename', data.title)
      } else if (data.type === 'onlyoffice-saveas') {
        if (data.ok && data.buffer) {
          this.$emit('save-as', new Blob([data.buffer], { type: this.mimeOf(data.fileType) }), data.fileType)
        } else {
          this.$emit('save-as-error', data.error || '另存为失败')
        }
      }
    },
    handleSaved(data) {
      const pending = data.requestId && this.pendingSaves[data.requestId]
      if (pending) {
        clearTimeout(pending.timer)
        delete this.pendingSaves[data.requestId]
        if (data.ok && data.buffer) {
          pending.resolve(new Blob([data.buffer], { type: this.mimeOf(data.fileType) }))
        } else {
          pending.reject(new Error(data.error || '保存失败'))
        }
        return
      }
      // 无匹配请求 = 编辑器自动保存触发，后台静默上传
      if (data.ok && data.buffer) {
        this.$emit('autosave', new Blob([data.buffer], { type: this.mimeOf(data.fileType) }), data.fileType)
      }
    },
    mimeOf(fileType) {
      const map = {
        docx: 'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
        xlsx: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
        pdf: 'application/pdf'
      }
      return map[(fileType || '').toLowerCase()] || 'application/octet-stream'
    },
    // 等待文档真正打开（onDocumentReady），最长 timeout 毫秒
    whenDocumentReady(timeout = 20000) {
      if (this.documentReady) return Promise.resolve()
      return new Promise((resolve, reject) => {
        const start = Date.now()
        const check = () => {
          if (this.documentReady) {
            resolve()
          } else if (Date.now() - start > timeout) {
            reject(new Error('编辑器加载超时'))
          } else {
            setTimeout(check, 120)
          }
        }
        check()
      })
    },
    /**
     * 触发 OnlyOffice 保存当前文档（onSaveDocument），返回编辑后的文件 Blob。
     * @param {String} format 目标格式（docx/xlsx）
     * @returns {Promise<Blob>}
     */
    async save(format) {
      const frame = this.$refs.frame
      if (!this.frameReady || !frame || !frame.contentWindow) {
        throw new Error('编辑器未就绪')
      }
      await this.whenDocumentReady()
      const requestId = `save-${++this.saveSeq}`
      return new Promise((resolve, reject) => {
        const timer = setTimeout(() => {
          delete this.pendingSaves[requestId]
          reject(new Error('保存超时，请确认已有修改后重试'))
        }, 20000)
        this.pendingSaves[requestId] = { resolve, reject, timer }
        frame.contentWindow.postMessage(
          { type: 'onlyoffice-save', requestId, format: format || this.fileType || undefined },
          '*'
        )
      })
    },
    onFrameLoad() {
      // iframe 已就绪（兜底，正常走 onlyoffice-ready 握手）
      if (this.frameReady) this.pushConfig()
    },
    buildConfig() {
      const cfg = {
        document: {
          url: this.url,
          title: this.title || undefined,
          fileType: this.fileType || undefined,
          key: this.docKey || undefined,
          permissions: {
            edit: this.mode !== 'view',
            download: true,
            print: true
          }
        },
        documentType: this.documentType || undefined,
        editorConfig: {
          mode: this.mode === 'view' ? 'view' : 'edit',
          lang: 'zh-CN'
        },
        height: '100%',
        width: '100%'
      }
      return this.deepMerge(cfg, this.config)
    },
    pushConfig() {
      if (!this.frameReady || !this.url) return
      const frame = this.$refs.frame
      if (!frame || !frame.contentWindow) return
      // 新文档开始加载，重置就绪状态，等待 iframe 回传 onDocumentReady
      this.documentReady = false
      frame.contentWindow.postMessage(
        { type: 'onlyoffice-config', docConfig: this.buildConfig() },
        '*'
      )
    },
    deepMerge(target, source) {
      const out = Array.isArray(target) ? target.slice() : { ...target }
      Object.keys(source || {}).forEach(key => {
        const sv = source[key]
        if (sv && typeof sv === 'object' && !Array.isArray(sv) && typeof out[key] === 'object') {
          out[key] = this.deepMerge(out[key], sv)
        } else {
          out[key] = sv
        }
      })
      return out
    }
  }
}
</script>

  display: block;
```

组件 API 一览：

| Props | 说明 |
|-------|------|
| `url` | 文档地址（必传） |

| `mode` | `edit` / `view` |
| `height` | 容器高度，数字或 CSS 值 |

| Events | 说明 |
|--------|------|
| `ready` / `document-ready` | iframe 就绪 / 文档打开完成 |

## 6. 支持的文件格式

| documentType | 编辑 | 预览/转换 |
|--------------|------|-----------|
| `word` | docx, odt, txt, rtf | doc, dot, dotx, epub, fb2, html, mht, xml 等 |
| `cell` | xlsx, ods, csv | xls, xlt, xltx, xlsm, fods 等 |
| `slide` | pptx, odp | ppt, pot, potx, pps, ppsx, fodp 等 |
| `pdf` | pdf（注释、表单填写、文本编辑、页面管理） | oxps, xps, djvu |

保存/导出时 `format` 传目标后缀即可实现格式转换（如 doc → docx、docx → pdf），转换完全在浏览器内由 `x2t.wasm` 完成。

## 7. 常见问题

**Q: `document.url` 有什么要求？**
文档由浏览器直接 `fetch`，跨域时目标服务需允许 CORS；本地文件可先 `URL.createObjectURL(file)` 生成 blob URL 传入。

**Q: 打开的还是旧内容 / 修改没生效？**
`document.key` 命中了编辑器缓存。文档内容变化后必须更换 key（如拼上版本号或更新时间）。

**Q: URL 参数方式报错或加载失败？**
docConfig 太长超出 URL 限制，改用 postMessage 方式（方式三）。

**Q: 触发保存后一直超时？**
确认 iframe 已收到 `onlyoffice-document-ready`（WASM 首次加载较慢）；检查控制台是否有字体/资源 404。

**Q: 用户点编辑器里的「下载」按钮没反应？**
默认 `streamFallback: 'autosave'` 会把流交给宿主而非下载。如需保留浏览器下载行为，配置里传 `streamFallback: 'download'`。

**Q: 能协同编辑吗？**
不能。离线版没有协同服务器，是单人编辑模型。多人场景需业务侧自行做互斥（如打开前加锁）。
